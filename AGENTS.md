# pi-config ‚Äî Agent Instructions

Personal [pi](https://github.com/earendil-works/pi-coding-agent) agent configuration.

## What this repo is

A dotfiles-style config repo for the **pi** coding agent. Lives at `~/.pi/agent/` (this directory). All config edits here take effect after `/reload` in pi.

## Setup

```bash
# Clone the repo to your pi agent directory
git clone <remote-url> ~/.pi/agent
```

After editing any config file ‚Üí run `/reload` in pi.

## Tracked vs excluded files

**Tracked** (edit these):

- `keybindings.json` ‚Äî keyboard shortcuts
- `decorated-pi.json` ‚Äî module toggles (safety, lsp, smart-at)
- `mcp-onboarding.json` ‚Äî MCP server onboarding state
- `extensions/*.ts` ‚Äî custom TypeScript extensions
- `themes/*.json` ‚Äî UI themes
- `prompts/`, `skills/` ‚Äî stubs for future content

**Excluded** (`.gitignore` ‚Äî do NOT commit):

- `settings.json` ‚Äî provider/model/theme/user prefs
- `auth.json` ‚Äî API keys
- `npm/` ‚Äî installed npm packages (runtime)
- `sessions/`, `run-history.jsonl` ‚Äî runtime data
- `models.json`, `mcp-cache.json` ‚Äî generated
- `.pi-lens/` ‚Äî LSP cache
- `extensions/app_data/` ‚Äî extension runtime data

## Key config files

- **`settings.json`** ‚Äî default provider (`opencode-go`), default model (`deepseek-v4-flash`), enabled models, theme (`astrodark`), subagent model overrides, memory/consolidation config, runtime-loaded npm packages list
- **`models.json`** ‚Äî provider definitions (opencode-backup, crof, sumopod, freemodel) with model IDs and API keys. Generated by scripts in the [dotfiles](https://github.com/iam/dotfiles) repo.
- **`decorated-pi.json`** ‚Äî `safety: true`, `lsp: false`, `smart-at: true`. LSP module disabled in favor of `pi-lens` (see Gotchas).
- **`themes/astrodark.json`** ‚Äî current active theme (set via `settings.json.theme`)

## Extensions

Three custom `.ts` extensions in `extensions/`:

- **`custom-footer.ts`** ‚Äî compact status bar footer (branch, model, tokens, cost). Toggle with `/footer` command.
- **`pi-info.ts`** ‚Äî system info viewer. Usage: `/info`, `/info all`, `/info tools`, `/info models`, `/info extensions`, `/info commands`, `/info context`
- **`youtube-transcript/`** ‚Äî fetch YouTube video transcripts and inject them into the conversation (directory-based extension with own `package.json`). Usage: `/youtube-transcript <video-url-or-id>` fetches the transcript and passes it to the agent as a user message. Registers the `youtube_transcript` tool for LLM use.
- **`ask-user.ts`** ‚Äî lean interactive `ask_user` tool (lean rebuild of `pi-ask-user` npm package). Registers the `ask_user` tool for decision gates. Supports single-select from options and freeform text input. Skill file at `extensions/ask-user.skill/` is registered via `resources_discover` event.

Extensions are auto-discovered by pi: any `.ts`/`.js` file or directory with `index.ts`/`index.js`/`package.json` in `~/.pi/agent/extensions/` or `<project>/.pi/extensions/`.

## npm packages

Managed in `npm/package.json` (private, gitignored). Local dependencies include: decorated-pi, pi-lens, pi-subagents, pi-mcp-adapter, pi-simplify, pi-total-recall, pi-web-access, pi-ask-user, pi-fzf, pi-image-preview, @plannotator/pi-extension.

Extensions with their own npm dependencies (e.g. `youtube-transcript/`) use a directory pattern with their own `package.json` and `node_modules/`.

## Runtime-loaded packages (settings.json)

pi loads only these packages at runtime (from `settings.json.packages` array): pi-web-access, pi-mcp-adapter, pi-total-recall, decorated-pi, pi-simplify, pi-lens, pi-subagents. (`pi-ask-user` was replaced by the custom `extensions/ask-user.ts` extension.)

Packages in `npm/package.json` but absent from this list (e.g. pi-fzf, pi-image-preview) are installed locally but not loaded by pi at startup.

## Gotchas

- `settings.json` is gitignored ‚Äî changes here are local only
- `models.json` is generated; do not edit manually. Regenerate via dotfiles repo scripts
- Scripts (e.g. `generate-models.js`) live in the [dotfiles](https://github.com/iam/dotfiles) repo, not here
- LSP module is disabled in decorated-pi (`lsp: false`) because pi-lens is the preferred LSP/diagnostics/auto-format provider. decorated-pi still provides smart @ completion, safety gates, and slash commands.
- Memory consolidation model (`opencode-backup/minimax-m2.5-free`) and `lessonInjection` mode are configured in gitignored `settings.json.memory` / `settings.json["pi-memory"]`.
